name: draft-commit-message
description: Use when asked to draft a Conventional Commits message based on a short summary of changes.
---

Draft a Conventional Commits message:

- Format: `type(scope): summary`
- Imperative mood ("Fix", "Add", "Remove")
- Summary <= 72 characters
- Add a short body only if needed (what/why, not a changelog)

Common types: `fix`, `feat`, `refactor`, `chore`, `test`, `docs`.
