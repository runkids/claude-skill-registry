# Domain Name Brainstormer

This skill generates creative, available domain names for your project by brainstorming names and checking real-time availability.

## When to Use This Skill

Use this skill when:
- Starting a new project or side project
- Launching a product or company
- Rebranding an existing service
- Need multiple domain options quickly
- Want to explore name variations

## Workflow

### 1. Understand the Project

Gather context:
- **What does it do?** Core functionality or purpose
- **Who is it for?** Target audience or users
- **What's the vibe?** Professional, playful, technical, creative?
- **Key words**: Important terms or concepts
- **Constraints**: Length preferences, must-include words

### 2. Brainstorming Strategy

Generate names using multiple approaches:

**Descriptive Names**:
- Directly describe what it does
- Combine function + industry
- Examples: CodeReview, DataSync, FlowTracker

**Portmanteau/Blends**:
- Combine two relevant words
- Examples: Snapchat, Pinterest, Instagram
- Your version: BidFlow, PropTracker, DealStream

**Modified Words**:
- Drop vowels: Flickr, Tumblr, Scribd
- Add prefixes: GetApp, UseThis, TryIt
- Add suffixes:Ify, ly, er

**Creative/Abstract**:
- Metaphors and concepts
- Examples: Airtable, Notion, Canva
- Your version: Compass, Catalyst, Horizon

**Acronyms**:
- Initials of key words
- Examples: AWS, IBM, API
- Make them pronounceable

**TLD Hacks**:
- Use TLD as part of name
- Examples: del.icio.us, bit.ly
- Your version: bid.io, track.dev

### 3. Name Criteria

Good domain names are:
- ✅ **Memorable**: Easy to recall
- ✅ **Pronounceable**: Can say it out loud
- ✅ **Spellable**: Easy to type correctly  
- ✅ **Short**: Ideally 6-12 characters
- ✅ **Brandable**: Sounds professional
- ✅ **Relevant**: Relates to product
- ✅ **Unique**: Distinctive from competitors

Avoid:
- ❌ Hyphens and numbers
- ❌ Common misspellings
- ❌ Trademark conflicts
- ❌ Negative connotations
- ❌ Hard to pronounce combinations

### 4. Domain Availability Check

For each brainstormed name, check availability across:

**Priority TLDs**:
- .com (always check first)
- .ai (for AI products)
- .dev (for developer tools)
- .io (for tech startups)
- .app (for applications)

**Secondary TLDs**:
- .co
- .tech
- .software
- .tools
- .studio

**Search Process**:
- Use domain availability API or web search
- Check each TLD systematically
- Note pricing if relevant
- Flag premium domains (high cost)
- Verify not trademarked

## Research Tools

**Free Domain Checkers**:
- Search: "domain availability check [domainname]"
- WHOIS lookup services
- Registrar search (GoDaddy, Namecheap, Google Domains)
- Instant Domain Search

**Additional Checks**:
- Social media handle availability (Twitter, Instagram, LinkedIn)
- USPTO trademark database search
- Google search for brand conflicts
- Domain history check (avoid spam/penalty domains)

## Tips for Best Results

1. **Generate 20+ options**: Don't stop at first few ideas
2. **Check variations**: Plural, different TLDs, with/without prefixes
3. **Consider SEO**: Keywords help discoverability
4. **Say it out loud**: Does it sound good?
5. **Test spelling**: Would people misspell it?
6. **Think long-term**: Will it work as you scale?
7. **Secure multiple TLDs**: Buy .com + branded TLD
8. **Move fast**: Good domains get taken quickly

## Advanced Strategies

**Competitive Analysis**:
- See what competitors use
- Identify naming patterns in industry
- Find gaps in naming space

**International Considerations**:
- Check meaning in other languages
- Avoid words that are offensive elsewhere
- Consider pronunciation globally

**Future Expansion**:
- Does name work for adjacent products?
- Can you build brand family with it?
- Room for tagline or descriptor?

**Social Handle Strategy**:
- Match domain with @handle when possible
- Acceptable: slight variations if core name works
- Reserve handles immediately after deciding
